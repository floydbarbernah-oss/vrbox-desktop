# VRBox Desktop

Виртуальный монитор ПК (VR BOX 2 + iPhone 11). На телефон ничего
не ставится, открывается страница в Safari. Экран ПК идёт на телефон по WebRTC, рисуется в стерео,
голова работает как мышь (управление взглядом). Личный проект, не продукт.

## Что умеет

- Стерео-рендер экрана ПК под VR-очки (Three.js `StereoEffect`).
- Трекинг головы по гироскопу плюс предсказание позы (меньше «плавания»).
- Управление взглядом: прицел в центре, наведение курсора на экран, gaze-меню (задержка взгляда около секунды).
- Мультиэкран: несколько виртуальных мониторов, выбор экрана взглядом.
- Резкость картинки (unsharp-шейдер) против «мыла» WebRTC. HUD с метриками: fps, битрейт, jitter.
- AR-фон с задней камеры (по отдельной кнопке).
- Изогнутый экран (шейдер `applyCurve`): края тянутся к зрителю, по бокам читаемее.
- Меню в стиле visionOS: матовое стекло, лежит на дуге и повёрнуто к зрителю, прячется за одну кнопку.
- Кап рендера 30 fps. Стерео рисует кадр дважды, это главный источник нагрева. На 30 поворот головы всё ещё плавный.

## Запуск (ПК, Windows)

```
py -3.14 -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt
.venv\Scripts\python pc\server.py
```

Сервер печатает две ссылки:

1. `http://IP:8080` — один раз установить и **доверить** сертификат (нужен для гироскопа):
   Настройки > Основные > Об устройстве > Доверие сертификатам, включить «VRBox Desktop».
2. `https://IP:8443/vr?k=ТОКЕН` — само приложение. Сохрани на рабочий стол iPhone.
   Токен обязателен: без него `/offer` не отдаёт поток и не пускает к управлению мышью,
   а `/monitors` не отдаёт раскладку экранов. Заход без токена возвращает 403.
   Полную ссылку сервер печатает только себе в консоль.

Телефон в той же Wi-Fi (лучше 5 ГГц). Не подключается — разреши Python во входящих в брандмауэре
Windows (порты 8080 и 8443).

Про уровень защиты, честно. Токен — заслон от случайного соседа по Wi-Fi, а не полноценная
авторизация: он не истекает, ходит в query-строке и лежит открытым в файле `.token`. Сервер даёт
управление мышью на ПК, поэтому поднимать его в чужой или публичной сети не стоит. Домашняя
Wi-Fi — сценарий, под который это писалось.

## Граница доступа

- HTTPS-корень `/` принимает только полную ссылку с токеном и затем переводит на `/vr?k=...`;
  без токена он возвращает 403 и не раскрывает значение из `.token`.
- `/vr`, `/offer` и `/monitors` проверяют один и тот же токен. Data channel управления мышью
  создаётся только внутри peer connection, прошедшего через авторизованный `/offer`.
- HTTP-страница настройки, `/cert` и локальные vendored-скрипты остаются публичными в пределах
  доступной сети: они не дают доступ к экрану или управлению.
- Ответы получают `Referrer-Policy: no-referrer`, `X-Content-Type-Options: nosniff` и запрет
  встраивания во frame. Строгий CSP пока не настроен: в HTML остаётся inline JavaScript.

Это защита домашнего LAN-прототипа, а не интернет-сервиса: нет пользователей, срока жизни токена,
rate limit и полноценного управления сессиями. Не выставляй порты 8080/8443 в интернет.

## Сертификат

Самоподписанный, лежит в `certs/` (в git не кладётся, там приватный `key.pem`). Пересоздать:
`python pc/make_cert.py` — LAN-адрес определяется сам, можно задать явно первым аргументом
(`python pc/make_cert.py 192.168.1.42`). Сертификат привязан к IP: при смене адреса перевыпустить
и заново доверить на телефоне.

## Проверка

Локальные проверки границы доступа не запускают сервер и не открывают сетевые сокеты:

```console
python -m unittest discover -s tests -v
```

Полный smoke-test видео требует Windows, установленных зависимостей, сертификата и реального
телефона в той же LAN.

## Структура

- `pc/server.py` — захват экрана (`mss`) → WebRTC (`aiortc`), data-channel управления мышью, раздача статики.
- `web/vr.html` — VR-клиент на Three.js: стерео, трекинг и предсказание позы, gaze-интерфейс, мультиэкран, HUD.
- `web/index.html` — страница установки сертификата (отдаётся по HTTP:8080).

## Статус

Рабочий прототип, обкатан вживую в очках. Стримится один монитор 1920,
поверх собран visionOS-подобный интерфейс.

Модель ввода устоялась: курсор наводится взглядом, а клик — физической BT-клавиатурой с тачпадом,
подключённой к ПК. В очках отдельная кнопка клика оказалась не нужна.
